# OKF Ask

`docs/okf/index.md`를 진입점으로 관련 노드를 탐색하고, 인용 근거와 함께 답변을 합성한다.

## 실행 흐름

단계 1 → 2 → 3 → 4 순서로 실행한다.

---

## 단계 1: 질문 확인

인자로 질문이 주어지면 그대로 사용한다. 없으면 사용자에게 묻는다.

```
OKF 위키에서 무엇을 찾고 싶으신가요?
```

---

## 단계 2: index.md로 관련 노드 탐색

`docs/okf/index.md`를 읽어 질문과 관련된 노드 후보를 추린다.

```bash
# index.md 존재 확인
ls docs/okf/index.md

# 질문 키워드로 노드 검색
grep -rl "<키워드>" docs/okf/
```

관련도가 높은 노드를 우선 읽는다.

---

## 단계 3: 답변 합성

읽은 노드 내용을 근거로 답변을 작성한다.

- 인용: 모든 주장에 출처 노드 파일명을 명시한다 (예: `[m13.md]`).
- 노드에 없는 내용은 추론하지 않는다. 없으면 "위키에 해당 정보가 없습니다"라고 밝힌다.

---

## 단계 4: 새 노드 저장 제안

답변 합성 과정에서 새로운 지식(기존 노드에 없는 연결·인사이트)이 나왔다면, 새 OKF 노드로 저장할지 사용자에게 제안한다.

저장을 원하면 `okf-authoring` skill 절차에 따라 노드를 작성한다.
